# `qc_results.csv` guideline

Generated by `master.py`. Each row represents one scan processed through the QC pipeline. Columns are grouped into five sections below.

---

## 1. Scan Information

| Column | Description |
|--------|-------------|
| `timestamp` |  Date and time the scan was processed |
| `scan_path` |  Absolute path to the input NIfTI file |
| `patient_id` | Filename (`.nii.gz` / `.nii` extension stripped) |

---

## 2. Artifact Detection 

Probability scores are in the range **[0, 1]**, where higher values indicate greater severity.

| Column | Description |
|--------|-------------|
| `artifacts_quality_passed` | `True` if the scan passed overall artifact QC |
| `artifacts_detected` | Pipe-separated list of artifact classes that exceeded threshold (e.g. `motion\|gibbs\|zipper`). Empty if none detected |
| `prob_clean` | Probability the scan is free of significant artifacts |
| `prob_motion` | Motion artifact severity score |
| `prob_noise` | Noise artifact severity score |
| `prob_ghosting` |  Ghosting artifact severity score |
| `prob_bias_field` | Bias field artifact severity score |
| `prob_gibbs` | Gibbs ringing artifact severity score |
| `prob_zipper` | Zipper artifact severity score |
| `iqm_motion_blur_score` |  Image quality metric: motion blur (higher = more blurred) |
| `iqm_snr` |  Image quality metric: signal-to-noise ratio (higher = better) |

---

## 3. Contrast Enhancement Detection 

Screens for gadolinium/contrast agent presence using heuristics on voxel intensity distributions.

| Column | Description |
|--------|-------------|
| `contrast_enhanced` |  `True` if gadolinium contrast enhancement is suspected |
| `contrast_vessel_ratio` | Ratio of bright vascular voxels relative to brain volume |
| `contrast_bright_voxel_fraction` | Fraction of brain voxels exceeding the brightness threshold used for contrast detection |

---

## 4. Registration QC

Only populated when a reference image is supplied via `--ref`. Quality checks using SSIM and NCC on brain-masked volumes.

| Column | Description |
|--------|-------------|
| `coreg_flag` | Overall registration QC flag: `GREEN` (both metrics pass), `YELLOW` (one fails), `RED` (both fail) |
| `coreg_ssim` | Structural Similarity Index (SSIM) between reference and registered image. Range [−1, 1]; higher is better. Default pass threshold: **0.70** |
| `coreg_ncc` | Normalised Cross-Correlation (NCC) between reference and registered image. Range [−1, 1]; higher is better. Default pass threshold: **0.80** |
| `coreg_ssim_passed` | `True` if `coreg_ssim` ≥ threshold |
| `coreg_ncc_passed` | `True` if `coreg_ncc` ≥ threshold |

### Flag interpretation

| Flag | Meaning |
|------|---------|
| `GREEN` | Both SSIM and NCC passed — registration acceptable |
| `YELLOW` | One metric failed — review recommended |
| `RED` | Both metrics failed — registration likely poor |

---

## 5. Metadata QC 

Sanity checks on image geometry and intensity, independent of the model.

| Column | Description |
|--------|-------------|
| `metaqc_status` | `pass` or `fail` |
| `metaqc_reasons` | Pipe-separated list of reasons if `metaqc_status` is `fail`. Empty on pass |
| `metaqc_foreground_fraction` | Fraction of voxels classified as foreground (brain + skull). Expected range roughly 0.10–0.35 |
| `metaqc_intensity_mean` | Mean voxel intensity within the foreground mask |
| `metaqc_intensity_std` | Standard deviation of voxel intensity within the foreground mask |
| `metaqc_centroid_offset_mm` | Distance (mm) between the image centroid and the field-of-view centre. Large values may indicate off-centre positioning |
| `metaqc_metadata_status` | `pass` or `fail` — checks NIfTI header validity (voxel size, orientation, etc.) |

## Overall QC decision logic

A scan is considered to have **passed** when all of the following are true:

- `artifacts_quality_passed` is `True`
- `metaqc_status` is `pass`
- `metaqc_metadata_status` is `pass`
- `coreg_flag` is `GREEN`
